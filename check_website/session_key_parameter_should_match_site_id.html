Check Website API

Session key parameter should match site id

GET /api/v1/sites/:id

All enrolled sites are scoped to particular user session. Server raises error if provided session key doesn't match site id

Parameters

Name Description
key Unique client key
id Website ID to check for state ("processing", "safe", "malicious")

Request

Route

GET /api/v1/sites/57736ace6a122f44e241dc30?key=57736ace6a122f44e241dc2e

Query Parameters

key: 57736ace6a122f44e241dc2e

cURL

curl "http://localhost:9292/api/v1/sites/57736ace6a122f44e241dc30?key=57736ace6a122f44e241dc2e" -X GET

Response

Headers

Content-Type: application/json
Content-Length: 28
Vary: Origin
X-Content-Type-Options: nosniff

Status

404 Not Found

Body

{
  "message": "Site not found"
}
