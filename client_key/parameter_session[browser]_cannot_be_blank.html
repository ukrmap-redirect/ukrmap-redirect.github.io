Client Key API

Parameter session[browser] cannot be blank

POST /api/v1/sessions

If the server receives a request without parameter session[browser] it will return an error.

Parameters

Name Description
session[browser] Browser name - the only one of client metrics

Request

Headers

Content-Type: application/json

Route

POST /api/v1/sessions

Body

{"session":{"browser":""}}

cURL

curl "http://localhost:9292/api/v1/sessions" -d '{"session":{"browser":""}}' -X POST \
	-H "Content-Type: application/json"

Response

Headers

Content-Type: application/json
Content-Length: 76
Vary: Origin
X-Content-Type-Options: nosniff

Status

422 Unprocessable Entity

Body

{
  "message": "Browser can't be blank",
  "errors": {
    "browser": [
      "can't be blank"
    ]
  }
}
